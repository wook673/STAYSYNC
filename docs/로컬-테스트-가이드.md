# staySync 확장 — 로컬 테스트 가이드

**Python·DB 설치 없이** `npm run dev`만으로 확장 ↔ 백엔드 왕복을 직접 테스트합니다.
(Next.js의 mock API가 백엔드 역할을 대신합니다.)

---

## 준비 (한 번만)

### 1. dev 서버 실행
```bash
cd C:\Users\User\staySync\frontend
node node_modules/next/dist/bin/next dev
```
→ http://localhost:3000 떠 있으면 OK

### 2. 확장 설치 (개발자 모드)
1. Chrome 주소창 → `chrome://extensions`
2. 오른쪽 위 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드** → 폴더 선택:
   ```
   C:\Users\User\staySync\extension
   ```
4. 목록에 **"staySync 연결 도우미"** + 🧩 아이콘 표시되면 성공

### 3. staySync 로그인 (JWT 발급)
1. http://localhost:3000/auth/login 접속
2. 아무 이메일·비밀번호 입력 → 로그인 (mock)
3. → 대시보드 진입 (이 탭은 **열어둔 채로** 두세요. 확장이 여기서 로그인 토큰을 읽습니다)

---

## 테스트 A · UI·상태 감지 (계정 없이 가능)

1. 브라우저 우측 상단 **🧩 staySync 아이콘** 클릭 → 팝업 열림
2. **확인 포인트:**
   - [ ] 5개 플랫폼(33m2·엔코스테이·리브애니웨어·자리톡·직방)이 보인다
   - [ ] 각 플랫폼이 **"플랫폼 로그인 필요"** 로 표시된다 (아직 로그인 안 했으니 정상)
   - [ ] staySync 로그인을 했으면 상단 경고가 사라진다

> 여기까지 보이면 **확장 로드 + UI + staySync 연동**이 정상입니다.

---

## 테스트 B · 실제 연결 왕복 (본인 플랫폼 계정 사용)

본인이 계정을 가진 플랫폼(예: **33m2** 또는 **직방**)으로 테스트합니다.

1. 새 탭에서 해당 플랫폼에 **내 계정으로 로그인**
   - 33m2: https://web.33m2.co.kr
   - 직방: https://www.zigbang.com
2. 다시 **🧩 팝업** 열기 → 새로고침
3. **확인 포인트:**
   - [ ] 그 플랫폼 상태가 **"로그인 감지됨 — 연결 가능"** 으로 바뀐다
   - [ ] **`연결`** 버튼이 활성화된다
4. **`연결`** 클릭
5. **확인 포인트:**
   - [ ] 팝업 하단에 **"연결 완료!"** 초록 메시지가 뜬다
   - [ ] (Mock 백엔드가 토큰 수신을 확인했다는 의미 — 왕복 성공 ✅)

## 테스트 C · 실제 플랫폼 예약 연동 확인 ★

실제 사용처럼 플랫폼 예약이 staySync로 들어오는지 확인합니다.
확장은 **선택자 → 휴리스틱(날짜 2개 자동 탐지)** 순으로 예약을 찾으므로,
선택자가 안 맞아도 대부분 자동으로 잡힙니다.

1. 플랫폼(예: 33m2)에 **내 계정으로 로그인** → **예약 목록/캘린더 화면**으로 이동
2. F12 → **Console** 탭을 연다
3. 자동 동기화 로그 확인:
   - ✅ `[staySync] 휴리스틱으로 N건 추출` + `동기화: {ok:true, added:N}` → **연동 성공!**
   - ⚠️ `예약을 찾지 못했습니다` → 아래 진단 실행
4. **진단 도구** (무엇이 감지되는지 직접 확인):
   ```js
   window.__staySyncDiagnose()
   ```
   - 선택자/휴리스틱 추출 건수 + 샘플 + 페이지 날짜 매칭 수가 출력됨
   - 0건이면: 예약 화면이 맞는지 확인하거나, **출력 샘플을 복사해 개발자에게 전달** → 정확한 선택자 반영

> 💡 **더 정확하게**: 실제 플랫폼 화면 구조를 직접 보고 선택자를 확정하면 이름·예약번호까지
> 완벽히 잡힙니다. 본인이 플랫폼에 로그인한 상태에서 진단 출력(또는 예약 한 건의 HTML)을
> 공유해주시면, 플랫폼별 `SELECTORS`를 정확히 맞춰드립니다.

## 잘 안 될 때

| 증상 | 원인 / 해결 |
|------|-------------|
| 팝업이 "staySync 먼저 로그인" 경고 | localhost:3000 로그인 탭이 닫혔음 → 다시 로그인 후 탭 유지 |
| `연결` 눌러도 무반응 | F12 Console에서 에러 확인. CORS면 manifest host_permissions 확인 |
| 상태가 "로그인 필요" 고정 | 플랫폼 탭에 실제 로그인됐는지 확인 후 팝업 새로고침 |
| 확장 수정 후 반영 안 됨 | `chrome://extensions`에서 확장 **새로고침(↻)** 클릭 |

> 코드를 고친 뒤에는 항상 `chrome://extensions` → 해당 확장의 **↻ 새로고침**을 눌러야 반영됩니다.

# Chrome 웹 스토어 게시 체크리스트

## 사전 준비
- [ ] **Chrome 웹 스토어 개발자 계정** 등록 (1회 등록비 $5)
  - https://chrome.google.com/webstore/devconsole
- [ ] **개인정보처리방침을 공개 URL로 호스팅** (`개인정보처리방침.md` 내용)
  - 예: GitHub Pages, staysync.kr/privacy
- [ ] manifest 내 `STAYSYNC_API`·도메인을 **운영 도메인**으로 교체 확인
  - `platforms.js`, `content-reservations.js`의 `localhost:8000` → `api.staysync.kr`

## 패키징
- [ ] 확장 폴더를 .zip으로 압축 (`pack.sh` 또는 수동)
  - 포함: manifest.json, *.js, popup.html, popup.css, icons/
  - 제외: README.md, .git, 임시파일
- [ ] zip 안에 **manifest.json이 최상위**에 있는지 확인 (폴더 한 겹 더 들어가면 거부됨)

## 에셋
- [ ] 아이콘 16/48/128 PNG (`icons/`) ✅ 생성됨
- [ ] 스토어 아이콘 128×128 ✅
- [ ] 스크린샷 1~5장 (1280×800 권장) — `docs/images/` 기반 재가공
- [ ] (선택) 프로모션 타일 440×280

## 개발자 대시보드 입력 (스토어-등록정보.md 참고)
- [ ] 이름 / 요약 / 상세 설명
- [ ] 카테고리: 업무 생산성
- [ ] 언어: 한국어
- [ ] 개인정보처리방침 URL
- [ ] 권한 사용 사유 (각 권한별)
- [ ] 데이터 안전(Data Safety) 양식
- [ ] 단일 목적 진술: "사용자의 숙박 플랫폼 예약을 staySync로 동기화"

## 심사 유의사항 (거부 사유 예방)
- [ ] **권한 최소화**: 쓰지 않는 권한 제거 (현재 cookies·scripting·storage·tabs·host)
- [ ] **원격 코드 없음**: 외부 `<script>`/eval 로딩 금지 (현재 없음 ✅)
- [ ] host_permissions가 광범위(`<all_urls>`)하지 않고 **지원 도메인으로 한정** ✅
- [ ] 단일 목적 원칙 준수 — 예약 동기화 외 기능 없음 ✅
- [ ] 개인정보처리방침이 실제 동작과 일치 ✅

## 제출 후
- [ ] 심사 보통 수 일 소요

- [ ] `docs/확장-설치-가이드.md`의 웹스토어 링크 갱신

## ⚠️ 법적/약관 검토 (게시 전 필수)
- 33m2·직방 등은 약관에서 자동 수집을 제한할 수 있음
- "사용자 본인 세션" 방식이라 방어 논거는 있으나 무해 보장 아님
- **공식 제휴(서면 동의) 병행 권고**, 게시 전 법무 검토 권장

# Chrome 웹 스토어 등록 정보 (복붙용)

게시 시 개발자 대시보드의 각 칸에 그대로 붙여넣을 수 있게 정리한 문서입니다.

---

## 기본 정보

| 항목 | 값 |
|------|-----|
| **이름 (Name)** | staySync 연결 도우미 |
| **카테고리** | 업무 생산성 (Workflow & Planning) |
| **언어** | 한국어 |
| **요약 (Summary, 최대 132자)** | 33m2·자리톡·엔코스테이·리브애니웨어·직방 예약을 내 로그인 세션으로 staySync에 연결해 한 화면에서 통합 관리하고 이중예약을 방지합니다. |

## 상세 설명 (Description)

```
staySync 연결 도우미는 여러 숙박 플랫폼의 예약을 하나로 모아 관리하는
staySync 서비스의 공식 보조 확장입니다.

자동 연동(iCal)을 제공하지 않는 플랫폼 — 33m2(삼삼엠투), 자리톡,
엔코스테이, 리브애니웨어, 직방 — 의 예약도 내 로그인 세션을 통해
staySync로 안전하게 가져옵니다.

■ 이렇게 동작합니다
1. 각 플랫폼에 평소처럼 내 계정으로 로그인합니다.
2. 확장 팝업에서 연결할 플랫폼의 '연결'을 누릅니다.
3. 로그인된 예약 화면의 정보를 읽어 staySync 캘린더로 동기화합니다.

■ 안전합니다
· 비밀번호를 저장하거나 전송하지 않습니다.
· 이미 로그인된 내 화면의 예약 정보만 읽습니다.
· 지원 플랫폼과 staySync 외 사이트에서는 동작하지 않습니다.

■ 이런 분께 좋습니다
· 여러 플랫폼에 숙소를 올려 운영하는 호스트
· 이중예약을 방지하고 싶은 단기임대 운영자
· 플랫폼마다 따로 캘린더를 확인하기 번거로운 분

staySync 계정이 필요합니다. (staysync.kr)
```

## 권한 사용 사유 (Permission Justifications)

> Chrome 웹 스토어 심사 시 각 권한마다 사유 입력이 요구됩니다.

| 권한 | 입력할 사유 |
|------|-------------|
| **cookies** | 사용자가 직접 로그인한 숙박 플랫폼의 세션 토큰을 읽어 예약 동기화를 인증하기 위해 사용합니다. 비밀번호는 다루지 않습니다. |
| **scripting** | 사용자가 로그인한 예약 화면에서 예약 날짜·식별자를 읽기 위해 콘텐츠 스크립트를 실행합니다. |
| **storage** | 플랫폼별 연결 상태를 로컬에 저장합니다. |
| **tabs** | 현재 열린 플랫폼/staySync 탭을 확인해 연결 상태를 판단합니다. |
| **host_permissions** | 지원 플랫폼(33m2·자리톡·엔코·리브애니웨어·직방) 및 staySync 도메인에서만 동작하기 위해 해당 도메인 접근이 필요합니다. |
| **원격 코드 미사용** | 모든 코드는 패키지에 포함되며 외부 스크립트를 불러오지 않습니다. |

## 데이터 安全 (Data Safety) 양식 응답

| 질문 | 응답 |
|------|------|
| 개인정보를 수집/사용하나요? | 예 — 예약 정보(날짜·식별자·게스트 표시명) |
| 데이터를 제3자와 공유하나요? | 아니요 |
| 데이터를 판매하나요? | 아니요 |
| 인증 정보(비밀번호)를 수집하나요? | 아니요 |
| 전송 시 암호화하나요? | 예 (HTTPS) |
| 사용자가 삭제 요청 가능한가요? | 예 (계정 삭제 시 함께 삭제) |
| 용도 | 앱 기능(예약 통합 표시·이중예약 감지) |

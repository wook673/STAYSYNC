# staySync 작업지시서

> Claude Code에서 이 파일을 읽고 작업을 이어받는다.
> 우선순위 순으로 정렬되어 있으며, 각 작업은 독립적으로 실행 가능하다.

---

## 프로젝트 한 줄 요약

한국 단기임대 호스트를 위한 **멀티 플랫폼 예약 통합 관리 SaaS**.
핵심 가치: 여러 플랫폼(에어비앤비·자리톡·33m2 등) 예약을 하나의 캘린더로 통합 + 이중예약 자동 감지.

## 작업 목록 (우선순위 순)

---

### TASK-1. 백엔드 E2E 검증 (최우선)

**목적:** 코드는 완성됐으나 실제 DB 연결·API 동작을 한 번도 검증하지 않았다.

**실행 방법:**
```bash
# 프로젝트 루트에서
docker-compose up --build

# 또는 백엔드만 (Python 환경 있을 때)
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

**검증 체크리스트:**
- [ ] `http://localhost:8000/docs` — Swagger UI 열리는지
- [ ] `POST /auth/register` — 회원가입 동작
- [ ] `POST /auth/login` — JWT 발급
- [ ] `POST /rooms/` — 숙소 생성
- [ ] `POST /rooms/{id}/connections` — iCal URL 등록
- [ ] `GET /calendar/events` — 예약 목록 반환
- [ ] `GET /calendar/conflicts` — 충돌 감지 동작

**예상 문제 및 해결:**
- DB 마이그레이션 누락 → `backend/app/db/database.py`에서 `Base.metadata.create_all()` 확인
- 환경변수 누락 → `backend/.env.example`을 `.env`로 복사 후 값 채우기
- asyncpg 연결 오류 → `DATABASE_URL` 형식: `postgresql+asyncpg://user:pass@localhost/dbname`

### TASK-2. Mock API → 실제 백엔드 전환

**목적:** 프론트엔드가 현재 Next.js Mock API Routes를 쓰고 있다. 실 백엔드로 전환한다.

**작업 파일:** `frontend/lib/api.ts`

```typescript
// 변경 전
const API_URL = process.env.NEXT_PUBLIC_API_URL || "http://localhost:3000";

// 변경 후
const API_URL = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000";
```

**추가 작업:**
- `frontend/.env.local` 생성:
  ```
  NEXT_PUBLIC_API_URL=http://localhost:8000
  ```
- Mock API Routes(`frontend/app/api/`)는 삭제하지 말고 유지 (데모 모드 필요시 재사용)
- 각 API 함수 호출이 실제 백엔드 응답 스키마와 맞는지 확인
  - 특히 날짜 포맷 (ISO 8601), 예약 상태값(`confirmed`/`blocked`/`cancelled`) 일치 여부

### TASK-3. 33m2 내부 API 엔드포인트 확정

**목적:** 시장 분석 기능의 실제 데이터 연결. 현재 mock 데이터 표시 중.

**사전 작업 (사람이 직접 해야 함):**
1. Chrome에서 `web.33m2.co.kr` 호스트 계정 로그인
2. F12 → Network → Fetch/XHR 필터 → Preserve log 체크
3. 지도/검색 화면으로 이동 → 매물 목록 XHR 캡처
4. 특정 매물 상세 진입 → 캘린더 XHR 캡처
5. 캡처 결과를 아래에 기입

**코드 반영 위치:** `extension/market-analysis.js` 상단

```javascript
const M33_ENDPOINTS = {
  mapSearch: "https://web.33m2.co.kr/???",   // ← 여기 교체
  calendar: (id) => `https://web.33m2.co.kr/???/${id}/???`,  // ← 여기 교체
};
```

**응답 구조 파악 후 추가 수정:**
- `calcOccupancy()` 함수의 파싱 로직 — 실제 응답 키에 맞게 수정
- `searchM33Real()` 함수의 `rooms = data.???` 부분 — 실제 배열 키로 수정

**상세 가이드:** `docs/33m2-내부api-캡처가이드.md`

### TASK-4. 대시보드 "시장 분석" 페이지 구현

**목적:** 확장 팝업(340px)은 지도 보기에 한계. 대시보드에 풀스크린 지도 페이지 추가.

**아키텍처 결정:**
- staySync 대시보드 안에 통합 (별도 프로젝트 분리 안 함)
- 사유: 동일 사용자, 확장 세션 토큰 재사용, 자연스러운 기능 세트

**생성할 파일:**

#### 1) 프론트엔드 페이지
`frontend/app/dashboard/market/page.tsx`

```
레이아웃:
┌─────────────────────────────────────────┐
│ 좌측 사이드바 (280px)  │  우측 지도 영역 │
│                        │                 │
│ [주소/위치 입력]        │  Kakao Maps     │
│ [반경: 500m ▼]         │  풀스크린       │
│ [예약률: 70%+ ▼]       │                 │
│ [기간: 2개월 ▼]        │  마커 색상:     │
│ [분석하기 버튼]         │  🟢 ≥90%       │
│                        │  🟡 70~89%     │
│ ─────────────────       │  ⚪ <70%       │
│ 결과 리스트             │                 │
│ (스크롤 가능)           │                 │
└─────────────────────────────────────────┘
```

지도 라이브러리: **Kakao Maps SDK** (한국 주소 정확도 높음)
- 스크립트 로드: `https://dapi.kakao.com/v2/maps/sdk.js?appkey=YOUR_KEY`
- 환경변수: `NEXT_PUBLIC_KAKAO_MAP_KEY`

#### 2) 백엔드 프록시 라우터
`backend/app/routers/market.py`

```python
# 목적: 33m2 API 직접 호출 시 CORS 차단 → 백엔드가 프록시
# 엔드포인트:
#   GET /market/search?lat=&lng=&radius=
#   GET /market/calendar/{room_id}?days=
# 인증: Bearer JWT (staySync 로그인 필요)
# 33m2 호출: 사용자의 저장된 33m2 세션 토큰 사용
```

#### 3) 대시보드 사이드바 메뉴 추가
`frontend/components/layout/Sidebar.tsx` (또는 해당 레이아웃 파일)
- "시장 분석" 메뉴 항목 추가
- 아이콘: `📊` 또는 `TrendingUp` (lucide-react)
- 링크: `/dashboard/market`

**Mock 데이터 우선 구현:** 백엔드 연결 전에도 Mock으로 UI 완성 → 이후 실 API 연결

## 기술 스택 요약

| 영역 | 스택 |
|------|------|
| 프론트엔드 | Next.js 14 (App Router), TypeScript, Tailwind CSS |
| 상태관리 | Zustand, React Query |
| 캘린더 UI | FullCalendar v6 |
| 백엔드 | FastAPI, SQLAlchemy (async), asyncpg |
| DB / 캐시 | PostgreSQL 16, Redis 7 |
| 스케줄러 | APScheduler (15분 주기 iCal 폴링) |
| 결제 | TossPayments |
| 알림 | Kakao 알림톡 (Solapi) |
| 지도 | Kakao Maps SDK (예정) |
| 배포 | Docker / docker-compose |

## 코드 규칙

- UI 텍스트·주석: **한국어**
- 백엔드: async/await 일관 사용
- 타인의 계정/로그인 정보 코드에 포함 금지
- 확장 `IS_PROD` 플래그 3곳 항상 동기화
- Mock API Routes(`frontend/app/api/`)는 삭제하지 말고 유지

## 로컬 실행

```bash
# 프론트엔드만 (Mock API, Docker 불필요)
cd frontend && npm install && npm run dev
# → http://localhost:3000 / 아무 이메일·비번으로 로그인

# 풀스택
docker-compose up
# → 프론트 :3000 / 백엔드 :8000 / DB :5432
```
